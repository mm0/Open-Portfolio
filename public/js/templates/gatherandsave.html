Gatherandsave was launched in 2013 as an iOS App designed to make ride sharing a snap. The goal was for organizations such as schools, workplaces and local communities in general to connect people together and find rides or passengers nearby heading towards your destination.

As backend developer for this project, it was my task to setup the application logic to track users, locations, destinations and routers and provide ride search results for users to find people to ride with. This project required the use of mathematics using data provided by MapQuest's API to find compatible rides in an efficient manner. Caching of these calculated routes provided further efficiency to minimize the required amount of computationfor the computationally intensive algorithms required to make this possible.

This project utilized my API Builder in conjunction with my Query Engine Framework, in an advanced state to create data organized within a MySQL database and to provide a solid algorithmic structure to the project requirement.
