QueryEngine arose out of the need for creating a better way of interacting with a (My)SQL after noticing patterns in the way data is accessed and manipulated within SQL.

With there being several ORM frameworks for PHP, PEAR (PHP Extension and APplication Repository) was chosen as a source for a minimal ORM framework that is easy enough for any PHP developer familiar with writing queries already, to use for interacting with a MySQL DB in an explicit, readable, and easy to use manner. After becoming familiarized with this ORM framework, it became evident that many if not all actions related to a Database were reusable if modified in a simple way, ie. changing table name, fields for retrieving data from a table and organizing the result set in a preferred way.

This was motivation for creating a way of minimizing the need for writing a query or application logic over and over for every action on a DB. All that was required, was recognizing the patterns within similar application logic methods and defining them as a general algorithm where you simply plug in values such as data source, validation functions, filtering functions, to use for that specific algorithm. The result was QueryEngine. Simply Write a type of algorithm once, recognize what logic this and reuse this logic for any project or function.

This may seem complicated to many developers, but I am confident that this is an extreme time saver and efficient way of managing any application that deals with a database. SImply configure a function type, and then reuse this later in any project.
